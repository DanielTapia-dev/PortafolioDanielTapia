REDLAB es una red de laboratorios médicos clínicos en la ciudad de Latacunga actualmente cuenta con 3 sucursales.
El sistema de redlab cuenta con una arquitectura cliente-servidor por lo que se escogieron dos tecnologias para realizar esta labor, estas son: Angular para el front-end y Nodejs para el back-end, ademas se utilizo una plantilla PRO adquirida en el sitio de wrappixel.com con el fin de no cargar el sistema con funciones inecesarias de la plantilla se utilizo unicamente todos los estilos css de la plantilla y los componentes especificos que el sistema necesitaba.

FRONT-END

Este cuenta con varios modulos que realizan diferentes procesos algunos entrelazados entre si como las proformas y las facturas.

Dashboard

Este modulo es la pagina principal de todo el sitio, esta pensado para funcionar como panel de control mostrando gráficas o actividades pendientes del usuario que se conecta el mismo no fue terminado porque no estaba contemplado en los requerimientos del sistema, pero hace muy escalable al mismo.

SideBar

En el sidebar se puede acceder a todas las funcionalidades del sistema

BACK-END

Este funciona mediante express y una base de datos relacional en PostgreSQL, se creo una API Rest para conectar el Front-End con el Back-End, para realizar la facturacion electronica el mismo tiene almacenada la firma electronica del cliente y se conecta directamente con el servidore de facturacion electronica del SRI y actualmente se encuentra en fase de produccion, vale recalcar que el sistema esta desarrollado con Javascript puro a diferencia del Front-End que utiliza TypeScript.

FACTURACION ELECTRONICA

Este es el modulo más importante del sistema ya que automatiza por completo el proceso de facturación, a continuacion se muestra la pantalla principal desde donde el cajero puede facturar facilmente, como se puede ver existe un buscador de cedula, y botones que sirven para agregar un nuevo cliente, facturar como consumidor final o eliminar el cliente para asignar uno nuevo a la factura, más abajo se puede observar el buscador de productos con el cual se podrá ir ingresando productos a la factura.

Escogiendo la forma de pago

Una vez la factura esta realizada se elige la forma de pago en una ventana modal con un buscador de las formas de pago existentes.

Se puede pagar con varias formas de pago a la vez

PDF

Esta es la factura en formato pdf que se envia directamente al cliente, en este caso es una factura de pruebas pero el sistema ya se encuentra en producción.
